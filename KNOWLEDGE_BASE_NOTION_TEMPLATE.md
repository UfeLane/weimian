# 卫眠伴行 Notion 知识库模板

这个模板适合你现在的 Demo 阶段：

- `Notion` 负责人工维护知识
- 项目里的本地样本负责先跑演示
- 后面如果要接小模型或 `Notion API`，字段可以直接复用

## 1. 睡眠知识库

数据库名建议：`睡眠知识库`

字段：

| 字段名 | 类型 | 用途 |
| --- | --- | --- |
| `标题` | Title | 如“什么是失眠” |
| `分类` | Select | `sleep` |
| `问题` | Text | 用户可能会问的问题 |
| `答案` | Text | 面向用户展示的标准答案 |
| `关键词` | Multi-select | 如 `失眠` `入睡困难` `睡眠日记` |
| `来源机构` | Text | 如 `NHLBI, NIH` |
| `来源简称` | Text | 如 `NHLBI 失眠专题` |
| `来源链接` | URL | 官方来源地址 |
| `更新时间` | Date | 内容更新时间 |
| `可公开回答` | Checkbox | 是否适合 AI 直接引用 |
| `备注` | Text | 编辑说明 |

## 2. 药品知识库

数据库名建议：`药品知识库`

字段：

| 字段名 | 类型 | 用途 |
| --- | --- | --- |
| `药品名` | Title | 如“达卫可” |
| `英文名` | Text | `DAYVIGO` |
| `通用名` | Text | `lemborexant` |
| `厂商` | Text | `Eisai / 卫材` |
| `适应症` | Text | 核心适应症 |
| `推荐剂量` | Text | 起始剂量和最大剂量 |
| `规格` | Multi-select | `5 mg` `10 mg` |
| `剂型途径` | Text | `片剂 / 口服` |
| `重要警示` | Text | 风险提示 |
| `常见不良反应` | Text | 如嗜睡 |
| `说明书摘要` | Text | 面向 Demo 展示 |
| `来源机构` | Text | `FDA` / `DailyMed` |
| `来源链接` | URL | 官方链接 |
| `更新时间` | Date | 内容更新时间 |

## 3. 说明书片段库

数据库名建议：`说明书片段库`

字段：

| 字段名 | 类型 | 用途 |
| --- | --- | --- |
| `标题` | Title | 如“用法用量” |
| `药品关联` | Relation | 关联到 `药品知识库` |
| `章节类型` | Select | `适应症` `用法用量` `警示` `不良反应` |
| `摘要正文` | Text | Demo 展示文本 |
| `来源简称` | Text | 如 `FDA 标签 PDF` |
| `来源链接` | URL | 官方链接 |
| `更新时间` | Date | 内容更新时间 |

## 4. 问答知识库

数据库名建议：`问答知识库`

字段：

| 字段名 | 类型 | 用途 |
| --- | --- | --- |
| `问题` | Title | 标准问法 |
| `分类` | Select | `sleep` `medication` `profile` `followup` |
| `答案` | Text | 用于展示或检索 |
| `关键词` | Multi-select | 检索召回 |
| `是否安全敏感` | Checkbox | 例如加量、停药 |
| `来源机构` | Text | 官方来源或内部档案 |
| `来源简称` | Text | 展示短标签 |
| `来源链接` | URL | 来源地址 |
| `是否启用` | Checkbox | 是否启用到 Demo |

## 5. 用户内部档案库

这个库不是公开知识，而是“程序内部数据”。

数据库名建议：`用户演示档案`

字段：

| 字段名 | 类型 | 用途 |
| --- | --- | --- |
| `用户名` | Title | 演示用户 |
| `当前药品` | Relation | 关联 `药品知识库` |
| `提醒时间` | Text | 如 `22:30` |
| `起始日期` | Date | 用药起始日 |
| `周期天数` | Number | 如 `14` |
| `已用片数` | Number | 如 `6` |
| `剩余片数` | Number | 如 `8` |
| `补药提醒日` | Date | 如 `2026-07-02` |
| `有效期` | Date | 如 `2026-07-18` |
| `近7天睡眠摘要` | Text | 程序内状态 |
| `复诊关注点` | Text | 给 AI 联动回答 |
| `下次复诊时间` | Date | 演示复诊安排 |
| `当前目标` | Text | 例如缩短入睡耗时 |

## 推荐接法

先做 `本地 JSON + Notion 镜像维护`：

1. Demo 当前继续读取本地样本 JSON
2. 你在 Notion 里维护知识库
3. 后面再加一个同步脚本，把 Notion 数据拉回本地

这样最稳，不会一开始就被 API、鉴权、限流卡住。
